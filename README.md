## 重装之前，先弄清楚几件事

重装系统会清空当前硬盘上的全部数据。废话，但我见过有人忘了备份，重装完才想起来服务器上跑着 MySQL 数据库。

DMIT 后台目前提供一个免费快照功能。重装前先给当前状态打个快照是最省事的操作，万一重装出问题还能回滚。进后台找到对应 VPS，找 Snapshot（快照）那个选项，一键创建，等几分钟搞定。

另外要注意：DMIT 默认是 SSH 密钥登录，不是密码登录。重装完之后，新系统的密钥需要重新下载或者重新配置。这一步很多人卡在这，后面会详细说。

## DMIT 后台重装系统：分步操作

### 第一步：登录后台，进入 VPS 详情

打开 DMIT 管理面板，登录账号，点击顶部菜单 **Services → My Services**，找到你想重装的 VPS，点进去。

进入 VPS 详情页之后，你会看到一排操作选项：开机、关机、重启、重装系统、快照、VNC 等。

### 第二步：找到重装系统入口

找到 **Reinstall**（重装系统）按钮，点进去。

DMIT 提供的系统选择还挺全的：Debian（11、12）、Ubuntu（20.04、22.04）、CentOS/Rocky Linux 这些常见发行版都有，还有一些小众选项。根据你的用途选——跑 Web 服务推荐 Debian 12，习惯 Ubuntu 生态就选 Ubuntu 22.04，别纠结太久。

### 第三步：确认重装

选好系统后，页面会提示你确认操作，通常还需要你输入账号密码做二次验证。

确认之后，后台会自动开始重装。整个过程一般 3–10 分钟，期间 VPS 处于不可用状态，稍等即可。

### 第四步：重装完成后重新获取密钥

重装完成后，**不要直接去 SSH 连接**——大概率连不上，因为密钥变了。

回到 VPS 详情页，找到 **Download Key** 选项，下载新的密钥压缩包。解压后你会看到 `private_key` 和 `public_key` 两个文件夹。你用的是 `private_key` 里面的：
- `id_rsa.pem`——适用于 XShell、MobaXterm、Mac/Linux 终端
- `id_rsa.ppk`——PuTTY 专用

拿到密钥之后，SSH 连接时指定这个密钥文件，用户名是 `root`，就能进去了。

## 重装完不想每次用密钥？设置密码登录

DMIT 默认只支持密钥登录，Root 密码默认仅用于控制台访问，直接 SSH 是不行的。要开密码登录，需要进系统改配置。

进去之后执行：

bash
passwd  # 设置 root 密码
sudo sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin yes/g' /etc/ssh/sshd_config
sudo sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication yes/g' /etc/ssh/sshd_config
sudo service sshd restart
reboot


注意：改完之后必须走一个完整的关机再开机流程（不是重启），新密码才会真正生效。这个细节很多人忽略，改完密码直接 SSH，发现还是连不上，其实只是少了那一步重启。

## 如果官网系统列表没有你要的系统

DMIT 后台提供的系统选项涵盖了大多数主流发行版，但如果你有特殊需求——比如想装某个特定版本，或者想 DD 安装 Windows——那就得走脚本路线了。

这类场景通常是：先通过后台重装一个能用的 Linux（比如 Debian 12），SSH 进去之后，再用一键 DD 脚本覆盖安装目标系统。脚本操作有风险，建议提前创建快照，万一搞崩了还能恢复。

## FAQ

**Q：DMIT 重装系统会丢失数据吗？**
A：会。重装是全盘覆盖安装，当前所有数据都会被清空。操作前先创建快照或手动备份重要文件。

**Q：重装之后还能用原来的 SSH 密钥吗？**
A：不能。重装后需要重新从后台下载新的密钥文件，旧密钥会失效。

**Q：DMIT 支持安装 Windows 吗？**
A：官方后台的系统列表主要是 Linux 发行版，没有直接提供 Windows 镜像。如有需要可以用 DD 脚本方式自行安装，但操作门槛更高，建议提前做好快照。

**Q：DMIT 重装系统要收费吗？**
A：后台的系统重装本身是免费操作，不额外收费。

**Q：选哪个机房最合适？**
A：看你的主要用途。国内用户访问速度要求高选香港，性价比考虑优先选洛杉矶，日本 IP 有特殊需求或者想分散节点选东京。洛杉矶是大多数人的首选起点。
